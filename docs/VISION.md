# VISION — Bytoviy Serbian (рабочее название)

Меняется редко и только по слову владельца продукта. Агенты: читать перед работой, не редактировать без явной просьбы владельца.

## Проблема

Русскоговорящий человек живёт в Сербии. Учить грамматику не хочет. Хочет быстро получить готовые фразы и потренироваться перед конкретной бытовой ситуацией — прямо сейчас, перед походом в аптеку или управу.

## Решение

Telegram-бот. Выбираешь ситуацию → получаешь шпаргалку из 5–10 фраз (сербская кириллица + латиница, произношение русскими буквами, перевод) → по желанию тренируешься в ролевом диалоге с ботом (бот играет врача/продавца/чиновника, мягко поправляет ошибки по-русски).

## Сценарии (бытовые ситуации)

Аптека · Поликлиника/скорая · Рынок · Магазин · Управа (продление ВНЖ) · Разговор с соседом · Мастер на дом · Такси · Кафе · Парикмахерская · СТО · Банк · Почта

## Функционал v0

1. `/start` — приветствие, меню с кнопками: список сценариев + «Мне срочно».
2. Карточка сценария: 5–10 фраз (кириллица, латиница, произношение русскими буквами, перевод) + пометки «ложный друг» для слов, похожих на русские, но с другим значением.
3. «Потренироваться» в карточке — ролевой диалог 5–7 реплик: бот в роли по сценарию, пользователь отвечает на сербском, бот по-русски мягко поправляет. В конце — короткая оценка (что получилось, что подтянуть).
4. «Мне срочно» — сразу список сценариев, по нажатию голая шпаргалка без предложения тренировки.
5. Лог событий (какой сценарий, когда, сколько сообщений в тренировке, id пользователя) — для подсчёта спроса.

## Что проверяем (это MVP)

Есть ли спрос: уникальные пользователи, возвраты, какие сценарии открывают, доходят ли до тренировки. Скорость запуска важнее архитектурной чистоты.

## Не-цели

Не курс грамматики. Не универсальный переводчик. Не полировка кода.
