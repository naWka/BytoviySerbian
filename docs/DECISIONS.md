Python-бот удалён (остаётся в истории git). Отменяет решения про aiogram/long polling/FSM/Railway для бота.
- **2026-07-04 — Память обучения: SRS (интервальные повторения, SM-2-lite).** Каждая карточка (реплика/слово) хранит прогресс: оценки «не помню / трудно / помню / легко» → интервал следующего повтора. Экран «Учить» показывает то, что пора повторить. Хранение — AsyncStorage на телефоне.
- **2026-07-04 — Контент расширен до 2000+ единиц.** 24 сценария-подготовки (что услышишь + чем ответить) + 40 тематических колод слов (~42 слова). Черновики сгенерированы AI и вычитаны AI-корректором «как носитель», помечены «черновик, не проверено носителем-человеком». Генерация — через фоновый workflow (генерация → вычитка на каждый файл).
- **2026-07-04 — Задел под AI-репетитора (BS-13).** Сейчас без AI (спарринг отложен, нужен ключ). Код структурирован так, чтобы позже подключить адаптивные тренировки и ролевой диалог поверх SRS.
- **2026-07-04 — Постоянные ID у слов/реплик прямо в контенте (BS-14).** Сейчас id карточки вычисляется из позиции в JSON (`vo:<колода>:<индекс>`, `sc:<сценарий>:...`). Правка контента (вставка/удаление/перестановка слова в середине) сдвигает индексы → прогресс SRS съезжает на соседние карточки. Решение: у каждого слова/реплики свой стабильный `id` в JSON, прогресс привязан к нему. Правки контента больше не ломают отметки «знаю». Владелец почуял риск («почему нет ID»), подтвердил направление.
- **2026-07-04 — Выученные слова сворачиваются вниз (BS-15, выбор владельца).** В списке темы выученное («знаю ✓») не исчезает, а собирается в свёрнутую группу «Выучено ▸» в самом низу; тап — раскрыть. Отклонены: «прятать совсем» (владелец хочет видеть, что уже освоил) и «тускло+фильтр» (лишний переключатель). Плюс одно-тап кнопка «знаю ✓» прямо на карточке при листании (сейчас оценки только в режиме «Учить»).
- **2026-07-04 — ПИВОТ вкладки «Учить»: только слова (BS-17).** Владелец: «полностью уйдём от Ситуаций» в учёбе. Вкладка «Учить» работает только со словами из колод; сценарии остаются справкой в «Ситуациях» (и их тренировка — с экрана сценария), в очередь «Учить» они не попадают. Баннер «пора повторить» убран из «Ситуаций» — за повторения отвечает «Учить».
- **2026-07-04 — Механика отбора слов: «знаю / учить / позже» (BS-17).** Как в словарных приложениях (triage + bury в Anki): свайп вправо = «знаю» (сразу в выученные), влево = «учить» (в пул, повтор доступен сразу), вниз = «позже» (отложенные, отдельный экран возврата). Пачки по 10, слова перемешаны по темам (round-robin), кнопки дублируют свайпы. Хранение: `buried` в zustand-сторе.
- **2026-07-04 — SRS без изменений (SM-2-lite), сессия = просроченные из пула (BS-17).** Новые слова в сессию не подмешиваются — их источник теперь разбор. «Не помню» возвращает слово в конец текущей сессии. Экран `review/[mode]` остаётся для тренировки темы/сценария/сохранённого.
- **2026-07-04 — Дизайн-обновление (BS-17): тёплая палитра + анимации.** Фидбек владельца: «дизайн прям типичный». Уходим от шаблонного серо-синего: тёплая бумажная палитра с терракотовым акцентом (светлая и тёмная). Анимации на reanimated: свайп-колода с физикой и штампами, 3D-переворот карточки, конфетти на финише сессии, haptics; уважаем reduced motion. Без новых зависимостей. Дизайн-контекст для агентов — docs/PRODUCT.md и docs/DESIGN.md.
