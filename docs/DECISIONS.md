- **2026-07-04 — Постоянные ID у слов/реплик прямо в контенте (BS-14).** Сейчас id карточки вычисляется из позиции в JSON (`vo:<колода>:<индекс>`, `sc:<сценарий>:...`). Правка контента (вставка/удаление/перестановка слова в середине) сдвигает индексы → прогресс SRS съезжает на соседние карточки. Решение: у каждого слова/реплики свой стабильный `id` в JSON, прогресс привязан к нему. Правки контента больше не ломают отметки «знаю». Владелец почуял риск («почему нет ID»), подтвердил направление.
- **2026-07-04 — Выученные слова сворачиваются вниз (BS-15, выбор владельца).** В списке темы выученное («знаю ✓») не исчезает, а собирается в свёрнутую группу «Выучено ▸» в самом низу; тап — раскрыть. Отклонены: «прятать совсем» (владелец хочет видеть, что уже освоил) и «тускло+фильтр» (лишний переключатель). Плюс одно-тап кнопка «знаю ✓» прямо на карточке при листании (сейчас оценки только в режиме «Учить»).
- **2026-07-04 — ПИВОТ вкладки «Учить»: только слова (BS-17).** Владелец: «полностью уйдём от Ситуаций» в учёбе. Вкладка «Учить» работает только со словами из колод; сценарии остаются справкой в «Ситуациях» (и их тренировка — с экрана сценария), в очередь «Учить» они не попадают. Баннер «пора повторить» убран из «Ситуаций» — за повторения отвечает «Учить».
- **2026-07-04 — Механика отбора слов: «знаю / учить / позже» (BS-17).** Как в словарных приложениях (triage + bury в Anki): свайп вправо = «знаю» (сразу в выученные), влево = «учить» (в пул, повтор доступен сразу), вниз = «позже» (отложенные, отдельный экран возврата). Пачки по 10, слова перемешаны по темам (round-robin), кнопки дублируют свайпы. Хранение: `buried` в zustand-сторе.
- **2026-07-04 — SRS без изменений (SM-2-lite), сессия = просроченные из пула (BS-17).** Новые слова в сессию не подмешиваются — их источник теперь разбор. «Не помню» возвращает слово в конец текущей сессии. Экран `review/[mode]` остаётся для тренировки темы/сценария/сохранённого.
- **2026-07-04 — ПЕРЕРАБОТКА механики учёбы (BS-18), отменяет механику BS-17.** Фидбек владельца: «не нравится механика и логика», «переделай полностью по топовым методичкам». Отменяет решения BS-17 «механика отбора знаю/учить/позже» и «сессия = только просроченные». Новая модель по evidence-based методикам (testing effect, learning steps, load management, active production):
  - **Один поток** вместо двух шагов: очередь занятия = просроченные + новые (лимит/день), перемешаны по темам. Отдельный экран разбора (triage) убран. Выбор владельца.
  - **Активное вспоминание обязательно**: сначала вопрос → пытаешься вспомнить → «Показать» → ответ → оценка. Нельзя пометить «знаю», не увидев ответа (главная дыра BS-17).
  - **Две стороны, слово «дорастает»**: новые/свежие — узнавание (SR→RU); зрелые (reps≥2 и интервал≥~3 дней) — говорение (RU→SR). Направление выводится из зрелости карточки, отдельного хранения на сторону нет (проще). Провал на говорении роняет слово обратно в короткие повторы. Выбор владельца: тренировать узнавание + говорение.
  - **Учебные шаги для новых/забытых**: короткие внутрисессионные показы (≈1 мин / 10 мин) перед выходом на дни. «Не помню» возвращает слово в конец текущего занятия.
  - **Дневной лимит новых** (по умолчанию ~10/день) для управления нагрузкой; счётчик newByDay в stats.
  - **«Позже» → «Убрать слово»**: bury переосмыслен как suspend (убрать из учёбы по одному, вернуть из отдельного списка), не как режим массового разбора.
- **2026-07-04 — График повторений (BS-18): учебные шаги + улучшенное SM-2-ядро; FSRS — на потом.** FSRS — текущий топ (дефолт Anki), но его преимущество раскрывается на истории повторений (оптимизация параметров), которой у нового пользователя нет; полноценный FSRS непропорционален MVP. Сейчас главный прирост эффективности дают активное вспоминание + учебные шаги + говорение + лимит новых, а не замена SM-2. Решение: оставить SM-2-ядро с настроенными константами + добавить учебные шаги; полноценный FSRS (с дефолтными параметрами) — отдельный будущий апгрейд, если понадобится.
- **2026-07-04 — Дизайн-обновление (BS-17): тёплая палитра + анимации.** Фидбек владельца: «дизайн прям типичный». Уходим от шаблонного серо-синего: тёплая бумажная палитра с терракотовым акцентом (светлая и тёмная). Анимации на reanimated: свайп-колода с физикой и штампами, 3D-переворот карточки, конфетти на финише сессии, haptics; уважаем reduced motion. Без новых зависимостей. Дизайн-контекст для агентов — docs/PRODUCT.md и docs/DESIGN.md.
- **2026-07-04 — Колода «Глаголы» со спряжением (BS-16): формат данных и показ таблицы.** Новая колода `content/vocab/glagoli.json` (id `glagoli`, 40 глаголов), встроена в общий раздел «Слова» без отдельного экрана — использует тот же экран колоды (`deck/[id].tsx`) и строку `WordRow`. Спряжение хранится прямо у слова как **массивы строк**: `present` (6 форм: ja/ti/on-ona/mi/vi/oni) и `past` (4 формы: он/она/они-м/они-ж, т.е. radio/radila/radili/radile). Каждая форма — строка `"кириллица · latin"` (обе графики в одной ячейке, как везде в приложении, но компактно — одна строка на лицо). Примеры глагола — массив `examples[{sr,ru,when}]` (1–2 шт.), где `when` = короткое «когда так говорят». **Почему так:** (1) массивы фиксированной длины проще всего править и ревьюить диффом, не плодят 10+ отдельных полей; (2) подписи лиц/родов держим в коде (`PRESENT_LABELS`/`PAST_LABELS`), а не в контенте — контент чище; (3) таблица показывается в раскрытой строке слова (тап по глаголу) двумя мини-таблицами «Настоящее время» и «Прошедшее время» — свёрнутая карточка остаётся лаконичной (инфинитив+произношение+перевод), читается с одного взгляда, а спряжение — по запросу. Совместимость: загрузчик берёт `examples[0]` как основной пример (`exampleSr/exampleRu`), поэтому глагол корректно показывается и в карточке занятия/повторения (FlipWordCard) без спец-кода. Контент — черновик AI + AI-вычитка «как носитель» (пометка в поле `_note` файла), спряжения выверены вручную; не проверено носителем-человеком.
