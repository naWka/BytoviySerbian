# BACKLOG — задачи

Правила:
- Задачи создаёт Product Manager из хотелок владельца. Разработчики сами задачи не придумывают.
- Статусы: `idea` (нужны ответы владельца) → `ready` (можно брать) → `in-progress` (укажи, кто взял) → `done` (+ дата).
- Разработчик берёт верхнюю `ready`, если владелец не указал другую.
- Критерии приёмки — только то, что владелец может проверить руками в Telegram.
- Нумерация сквозная: BS-N.

## BS-1: Скелет бота — `ready`

Как пользователь, хочу открыть бота в Telegram и получить готовые фразы для трёх ситуаций.

Объём: /start с приветствием и меню, карточки 3 тестовых сценариев (аптека, такси, кафе) по 5–10 фраз из data/scenarios.json, кнопка «Мне срочно» (список сценариев → голая шпаргалка). Без AI. Кнопка «Потренироваться» есть, но отвечает «скоро». .env.example, requirements.txt, инструкция запуска в CLAUDE.md.

Критерии приёмки:
- [ ] Владелец кладёт токен в .env, запускает одну команду — бот отвечает в Telegram
- [ ] /start показывает меню: 3 сценария + «Мне срочно»
- [ ] Нажатие сценария — карточка: кириллица, латиница, произношение русскими буквами, перевод, пометки «ложный друг»
- [ ] «Мне срочно» — список сценариев, по нажатию шпаргалка без кнопки тренировки

Блокер: токен от @BotFather (нужен от владельца).

## BS-2: Ролевая тренировка через Claude — `ready` (после BS-1)

Как пользователь, хочу потренировать диалог перед походом, чтобы не растеряться вживую.

Объём: кнопка «Потренироваться» запускает диалог 5–7 реплик; бот играет роль по сценарию, пользователь пишет на сербском, бот по-русски мягко поправляет и продолжает; в конце короткая оценка. ANTHROPIC_API_KEY в .env.

Критерии приёмки:
- [ ] «Потренироваться» в карточке аптеки — бот начинает диалог в роли аптекаря
- [ ] Ответ с ошибкой на сербском — бот мягко поправляет по-русски и продолжает
- [ ] Через 5–7 реплик — оценка: что получилось, что подтянуть
- [ ] После оценки можно вернуться в меню

## BS-3: Лог событий — `ready` (после BS-1)

Как владелец, хочу видеть, пользуются ли ботом, чтобы понять спрос.

Объём: SQLite-таблица events (user_id, тип события, сценарий, время, длина тренировки). Открытие сценария, «Мне срочно», старт/конец тренировки. Без дашбордов — просто таблица + пара готовых SQL-запросов в docs для «сколько уникальных, сколько вернулось».

Критерии приёмки:
- [ ] Владелец кликает бота → агент показывает выгрузку: события записались
- [ ] Есть ответ на вопросы: сколько уникальных пользователей, какие сценарии открывали
