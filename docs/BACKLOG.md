# BACKLOG — задачи

Правила:
- Задачи создаёт Product Manager из хотелок владельца. Разработчики сами задачи не придумывают.
- Статусы: `idea` (нужны ответы владельца) → `ready` (можно брать) → `in-progress` (укажи, кто взял) → `done` (+ дата).
- Разработчик берёт верхнюю `ready`, если владелец не указал другую.
- Критерии приёмки — только то, что владелец может проверить руками в Telegram.
- Нумерация сквозная: BS-N.

## BS-1: Скелет бота — `done` 2026-07-04 (ждёт проверки владельцем)

Как пользователь, хочу открыть бота в Telegram и получить готовые фразы для трёх ситуаций.

Объём: /start с приветствием и меню, карточки 3 тестовых сценариев (аптека, такси, кафе) по 5–10 фраз из data/scenarios.json, кнопка «Мне срочно» (список сценариев → голая шпаргалка). Без AI. Кнопка «Потренироваться» есть, но отвечает «скоро». .env.example, requirements.txt, инструкция запуска в CLAUDE.md.

Критерии приёмки:
- [ ] Владелец кладёт токен в .env, запускает одну команду — бот отвечает в Telegram
- [ ] /start показывает меню: 3 сценария + «Мне срочно»
- [ ] Нажатие сценария — карточка: кириллица, латиница, произношение русскими буквами, перевод, пометки «ложный друг»
- [ ] «Мне срочно» — список сценариев, по нажатию шпаргалка без кнопки тренировки

Токен получен, бот @Bytoviy_Serbian_bot, подключение проверено. Вердикт владельца: карточки v1 — «полная хрень», фразы слишком базовые для ЦА. Скелет остаётся, контент и формат карточек заменяются в BS-6 (пивот, см. VISION v2).

## BS-2: Спарринг с уходом от скрипта — `ready`, блокер: ANTHROPIC_API_KEY

Как пользователь, хочу отработать разговор, в котором собеседник ведёт себя как живой человек, чтобы не плыть в реальной ситуации.

Объём: кнопка «Спарринг» запускает диалог 5–7 реплик; бот в роли (аптекарь/чиновник/мастер) **сознательно уходит от скрипта**: встречные вопросы, повороты («этого нет в наличии», «не хватает документа»), разговорная речь. Пользователь пишет на сербском, бот по-русски мягко поправляет и продолжает. В конце разбор: что получилось, где поплыл, какие фразы из карточки пригодились бы.

Критерии приёмки:
- [ ] Спарринг в аптеке: бот-аптекарь задаёт встречный вопрос, а не ждёт реплик по списку
- [ ] Минимум один неожиданный поворот за диалог
- [ ] Ошибка в сербском — мягкая поправка по-русски, диалог продолжается
- [ ] В конце разбор: что получилось, где поплыл
- [ ] После разбора можно вернуться в меню

Блокер: ANTHROPIC_API_KEY от владельца (console.anthropic.com).

## BS-6: Карточки «второго слоя» — `in-progress` (агент основной сессии)

Как пользователь с годом жизни в Сербии, хочу подготовку к реальному разговору, а не разговорник для туристов.

Объём (пивот, см. VISION v2 и DECISIONS от 2026-07-04):
- Новый формат карточки: 👂 «Что услышишь» (типичные реплики собеседника, разговорные варианты, как среагировать) + 🗣 «Чем ответить» (фразы второго слоя).
- Демо-сценарии: аптека (глубокая), управа/ВНЖ, мастер на дом. Такси и кафе удалить.
- Отдельная карточка «🆘 SOS-фразы»: замедлить, переспросить, попросить написать.

Критерии приёмки:
- [ ] В меню: Аптека, Управа (ВНЖ), Мастер на дом, SOS-фразы, «Мне срочно»
- [ ] Карточка: сначала «что услышишь» с реакциями, потом «чем ответить»; фразы не из разговорника для туристов
- [ ] SOS-карточка открывается из меню
