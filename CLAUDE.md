# CLAUDE.md

Проект «Bytoviy Serbian» — мобильное приложение (Expo/React Native) для русскоговорящих в Сербии: подготовка к бытовым разговорам («что услышишь» + «чем ответить»), колоды слов, память обучения с интервальными повторениями (SRS). MVP для проверки спроса: скорость запуска и простота кода важнее архитектурной чистоты. (Раньше был Telegram-бот — заменён приложением, см. DECISIONS.md.)

## Разработка полностью через агентов

Владелец продукта НЕ читает код и НЕ отвечает на технические вопросы. Он описывает хотелки и оценивает результат («нравится / не нравится»). Перед любой работой прочитай:

1. [docs/VISION.md](docs/VISION.md) — что за продукт и зачем. Не редактировать без явной просьбы владельца.
2. [docs/PLAN.md](docs/PLAN.md) — этапы, текущий фокус.
3. [docs/BACKLOG.md](docs/BACKLOG.md) — задачи и статусы. Единственный источник «что делать».
4. [docs/DECISIONS.md](docs/DECISIONS.md) — принятые технические решения (append-only).

## Роли

- **Product Manager** (`.claude/agents/product-manager.md`) — общается с владельцем, превращает хотелки в задачи BACKLOG.md. Основная сессия по умолчанию ведёт себя как PM: владелец описал желание или фидбек → сформулируй задачу, запиши в бэклог, подтверди простыми словами.
- **Developer** (`.claude/agents/developer.md`) — берёт задачу `ready` из бэклога, реализует, обновляет статус, коммитит. Запускается, когда владелец говорит «делай» / «в работу» / «следующая задача».

## Правила для всех агентов

- Общение с владельцем: русский, простой язык, ноль жаргона. Формат отчёта: что сделано → что проверить руками (в приложении) → что дальше.
- Технические выборы принимай сам, фиксируй в DECISIONS.md (дата, решение, почему). Владельца спрашивай только о продукте.
- Изменился план → сначала PLAN.md и BACKLOG.md, потом код.
- Задача сделана → статус в BACKLOG.md (`done` + дата), локальный коммит (Conventional Commits, subject ≤50 символов).
- `git push` — только с явного разрешения владельца.
- Секреты только в `.env` (в .gitignore). `.env.example` с плейсхолдерами — коммитить.

## Стек

Expo SDK 57, React Native 0.86, expo-router (файловые табы), TypeScript. Цель поставки — **PWA** (web-экспорт через react-native-web, ставится на телефон, офлайн через service worker). Состояние — zustand; локально AsyncStorage/localStorage (офлайн-кэш), в облаке — **Supabase** (auth email+пароль + `profiles.state jsonb` с RLS, источник правды на пользователя, синк debounce-апсертом). Контент — JSON в `content/vocab/` (колоды слов), собирается в `content/index.ts` скриптом. Хостинг — GitHub Pages, деплой — GitHub Actions; своего бэкенд-сервера нет (браузер ходит в Supabase напрямую). Пивот в PWA — BS-22 (DECISIONS). Задел под AI-репетитора (BS-13) — на будущее.

## Команды

```bash
npm install                            # один раз и при смене зависимостей
npx expo start                         # запуск (QR в Expo Go на телефоне; --tunnel если не видит)
node scripts/build-content-index.mjs   # пересобрать индекс контента после правки content/*.json
npx tsc --noEmit                       # проверка типов
```

Код — в `src/` (экраны в `src/app/`). Целевое дерево — в [docs/PLAN.md](docs/PLAN.md).
